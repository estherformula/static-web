발표자 임수용 소속 개발팀

1 추진 배경 및 목적

AI 도구는 더 이상 개발 부서만의 것이 아닙니다. 최근 에이전트형 AI는 사람이 말로 지시하면 문서·엑셀·자료조사 같은 실무를 직접 수행해 결과물까지 만들어내는 수준에 이르렀습니다. 그러나 사내 비개발 직무 구성원 대부분은 이러한 도구를 처음 배우기가 쉽지 않습니다.

본 교육의 목적은 두 가지입니다. 첫째, 반복·수작업 중심 업무의 생산성을 끌어올리는 것. 둘째, 외부 의존 없이 구성원 스스로 AI를 다루는 역량을 사내에 내재화하는 것입니다. 한 번의 특강이 아니라, 구성원이 자기 업무에 직접 적용해보는 출발점을 만드는 데 초점을 둡니다.

사내 AI 활용 문화 정착

① 업무 생산성 향상

  • 난잡한 회의 메모 → 안건·액션아이템 자동 정리
  • 흩어진 엑셀 수만 행 → 함수 없이 통합·집계
  • 설문 자유응답 수백 건 → 핵심 인사이트 요약
  • 부서간 서로 다른 양식으로 문서변환의 자동화

② AI 역량 내재화

  • “어디까지 되나”가 아니라 “어디부터 써볼까” 마인드셋 전환
  • 실전형 프롬프트 작성요령 교육
  • 반복 작업을 사내 표준 명령으로 정착(예: 주간보고 자동화)
  • 교육 자료·노하우를 사내에 축적해 자생적 확산 기반 마련
본 세미나의 성공 기준은 “이해시켰는가”가 아니라 “세미나 직후 자기 업무 하나를 직접 맡겨보게 만들었는가”입니다.
측정 지표 — 세미나 후 2주 내, 참석자 1인당 최소 1건의 활용 사례를 사내 Q&A 채널에 공유(강제 X, 차기 세션 판단 기준).

4 프로그램 구성

발표 (15분)

  • 클로드코드란? — “해주는” AI 비서
  • 챗GPT와의 차이 = 왜 써야 하나
  • 에이전트의 강점: “박사학위 딴 신입사원”
  • 무엇을 맡길 수 있나 / 기본 사용 패턴
  • 왜 덜 틀리나 — 확인·검산의 습관

라이브 시연 (30분)

  • ① 회의록 정리 → ② 엑셀 통합 (고정 앞단 · 전 직무 공감)
  • ③ 파일·사진 정리 / ④ 설문 자유응답 요약 / ⑤ 보도자료 채널 변환 (보너스 슬롯)
  • ⑥ 매뉴얼·정책 지식베이스화 (심화 쇼케이스)
실전 사례는 비개발 직무가 바로 적용 가능한 것 중심. 개발팀 사례는 후순위로 간단 공유.

5 향후 확산 계획

  • 세미나 직후 ~2주, 가벼운 Q&A 채널(슬랙) 운영 — 참석자 막힘 방지 + 활용 사례 수집. 쌓인 질문이 차기 세션 기획의 1순위 재료.
  • 성공 지표(2주 내 1인 1건 공유) 달성률을 차기 세션 개최여부 판단 기준으로 활용
  • 개인/팀에서 AI 도입 시 필요한 경우 대면지원 계획
  • 희망하는 경우 부서별 맞춤 실습 세션(스킬·서브에이전트 등 심화 포함)
