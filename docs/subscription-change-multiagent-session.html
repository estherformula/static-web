1기획 검토 의뢰 — 블로커·누락 진단

사용자

구성변경 기능개선 초안([개요]/[as-is]/[to-be])으로 지나에게 검토시키고 ⓪폴더 생성 ①블로커 점검 ②부족한 부분 채워 md 저장.

▶ 사용자 프롬프트 전문
에스더팩의 구독상세화면에서 '구성변경' 기능이 있는데 기능 개선을 하기위해 기획을 진행할거야 아래 내용으로 계획을 짰는데 이 기준으로 지나에게 검토시키고 아래 순서로 일을 진행해 0. 스펙리뷰 폴더에 이 건을 진행할 폴더를 하나 만들고 1. 블로커가 있는지 점검 2. 언급되지않은 부족한 부분을 채워서 md파일로 저장 제목 : 1:1문의 기능을 이용한 구성변경 기능개선 [개요] - 구독 상세화면에서 사용자가 직접 구성변경을 요청할 수 있다 - 사용자가 구성변경을 누르면 채널톡이 열리면서 구독을 일시정지하게된다 - 채널톡의 워크플로우는 별도 솔루션에서 진행되므로 진행상태를 알 수 없다 - 사용자가 채널톡에서 액션을 수행하지 않으면 상담원에게 상담이 전달되지 않는다 - 이렇게 되면 구독은 일시정지 되었지만 구성변경 상담은 진행되지 않는 상태가 된다 - 신설된 1:1문의 기능을 이용하면 사용자가 문의를 등록완료하는 시점에 구독을 일시정지 시킬 수 있으니 좀비상태의 구독을 만들지 않아도 된다 [as-is] - 사용자가 구독상세 화면에서 구성변경 버튼 클릭 - 채널톡 워크플로우 오픈 (구성변경 = "780572") - 동시에 '구독일시정지 API' 를 호출하여 구독을 일시정지한다 [to-be] - 사용자가 구독상세 화면에서 구성변경 버튼 클릭 - 1:1문의 입력폼이 나타나며 구성변경 요청내용을 사용자가 입력할 수 있다 - 기존 1:1문의 입력폼과 다르게 '문의 유형', '연결 주문', '제목', '이미지 첨부'가 없다 - 필수입력 : '내용', '알림수단' - 문의등록 API 요청시 구독번호(subscription_number) 를 파라미터에 포함시킨다 - 등록완료시 '구성변경이 요청되었어요.\n완료되면 알려드릴게요.' 토스트가 노출 - 구독 일시정지는 해당 문의가 접수되는 순간 BE에서 처리한다 - 구성변경 1:1문의 유형 타입을 신규로 추가한다 (적절한 이름 제안) - 이 유형은 고객센터의 1:1문의에서는 입력할 수 없다 (선택지에 노출되지 않음)
메인 작업 폴더 생성 → 재활용 기반인 기존 1:1문의 기획서를 먼저 읽어 컨텍스트 확보 → 지나에게 검토 의뢰.
에이전트 · 지나 수행: 초안과 실제 출시 1:1문의 정책·구독 정책·iOS 가이드를 교차 검토. 블로커 3건 + 진행가능 우려 11건 + 결정필요 8건 도출.

▶ 메인이 Explore에게 보낸 지시 프롬프트 보기
EstherPack iOS 코드베이스에서 "구독이 일시정지(paused) 상태일 때 구성변경 버튼이 미노출(숨김/비활성)되는가?"를 코드 근거로 확인해줘(very thorough). 찾을 것: ①구독 상세 ViewController/View/ViewModel ②'구성변경' 버튼(한글/영문, 채널톡 워크플로우 코드 780572도 단서) ③버튼 isHidden/isEnabled 노출 분기가 구독 상태값과 연결됐는지 ④구독 상태 enum 정의(일시정지 해당값) ⑤일시정지 API·채널톡 오픈 로직. 보고: 결론(그렇다/아니다/조건부) + 근거 코드(파일:줄) + 상태 enum 값 + 노출 조건 전체. 추측 금지, 실제 파일·줄번호로.
메인 이 코드 사실을 지나에게 전달 → OPEN-6(중복방지)·OPEN-7(진입분기)이 "기존 노출 로직으로 이미 해결"임이 드러나 신규 구현 다수 제거. 범위가 크게 가벼워짐.
결과물 OPEN 8건 전부 CLOSED → 기획서·OPEN 트래커에 일괄 반영. 블로커 3건 중 실제 BE 신규작업은 1건으로 축소.

▶ 사용자 프롬프트 전문
시안이니까 지나가 적절하게 피드백해서 안나가 바로 시안 작업하게 해줘
에이전트 · 지나 수행: 디자인 미결 3건을 시안용으로 확정 — D-1 폼 상단 안내문 카피, D-2 대상 구독 요약 카드(팩 이름 단독·읽기전용), D-3 완료시트 단일 [확인]→구독상세 복귀.
에이전트 · 안나 수행: 지나 확정값으로 Figma에 구성변경 시안 6프레임 생성. 기존 1:1문의 v2 자산 클론 변형 + EstherPack 토큰 바인딩(off-token 0), 자가검수 게이트 통과.
▶ 메인이 안나에게 보낸 시안 착수 지시 프롬프트 보기
디자인 미결 3건이 지나 피드백으로 확정됐다. 이제 실제 구성변경 시안 작업에 착수해줘. D-1 안내문(확정 카피 2줄: "변경하실 구성을 자유롭게 적어주세요. / 요청을 접수하면 영양상담사가 확인 후 도와드려요."), D-2 구독 요약 카드(팩 이름 단독, 읽기전용, 구독번호 미노출), D-3 완료 바텀시트(단일 [확인]→구독상세 복귀, [내 문의 보기] 제거, 기기알림 켜기 조건부 유지). 폼 구성(위→아래): 네비 타이틀 "구성 변경 요청" → 안내문 → 대상 구독 카드 → 내용(카운터) → 알림수단 → 하단 [등록하기]. 전역 UI 규칙(색=토큰, 폰트=스타일, 공용 컴포넌트 재사용, SnapKit/Then, raw 금지) 준수, 기존 v2 시안 베이스 변형. Figma 작화 가능하면 그려라. 막히면 무엇이 필요한지 알려라.
결과물 · Figma 시안 (1차) 작성폼(활성/비활성), 수정폼, '구성변경' 배지, 완료시트(기본/알림권장) — 총 6프레임. (Figma 파일 "APP-UI-with-AI", 페이지 "1:1 문의")

5시안 피드백 반영 (3차례)

⑤-A 로직 주석 겹침 + 에이전트 규칙 보강
사용자

[로직] 코멘트가 화면과 겹침 → 프레임 아래로 배치 수정 + 안나 규칙에 영구 반영.

▶ 사용자 프롬프트 전문
디자인시안 피드백줄게 반영해줘 - [로직] 코멘트를 화면마다 달아놨는데 위치가 화면이랑 겹쳐있음. 프레임 사이즈가 큰 경우 제대로 프레임 아래에 배치되도록 수정하고 앞으로도 그렇게 하게 안나 규칙에 넣어줘
메인 ① 안나에게 현재 시안 주석 재배치 지시 ② 안나의 영구 규칙 파일(상시 강제 + 자가검수 게이트)에 "주석은 프레임 높이 아래 간격 두고 배치, 겹침 금지" 규칙 추가.
에이전트 · 안나 수행: 각 주석을 프레임 실제 높이(y+height) 아래 32px 간격으로 재배치, 겹치던 행은 아래로 이동. 겹침 검증 0건.
⑤-B 기존 화면 연결 + 상태 화면 + 화면 전이
사용자

기존 구독상세 연결점 + 내 문의내역(구성변경 건) + 문의상세(답변대기) 추가, 화면 전이(클릭→이동) 표현.

▶ 사용자 프롬프트 전문
아래 내용도 시안에 반영해줘 1. 기존화면과의 연결점도 시안에 함께 포함되었으면 좋겠어 기존 구독상세화면 - (Figma 링크) 2. 1:1문의 - 내 문의내역 화면에 구성변경 건이 존재하는 상황 화면 3. 1:1문의 - 문의상세 화면에 구성변경 건 상세화면 (답변대기중) 위 화면들 추가하고 화면에서 어디를 클릭하면 어느 화면으로 넘어가는지도 표현해줄 수 있어?
에이전트 · 안나 수행: 진입점(기존 구독상세, 외부 파일이라 스크린샷 참조본)·내 문의내역(S3', 구성변경 셀 포함)·문의상세(S4', 답변대기) 3프레임 추가 + 화면 전이 화살표 5개(탭 대상 라벨 포함). 겹침 0.
▶ 메인이 안나에게 보낸 지시 프롬프트 보기
구성변경 SC 그룹에 추가: ① 기존 구독상세 화면(외부 Figma 파일)을 진입점 참조로 배치 — '구성변경' 버튼이 출발점(ACTIVE에서만 노출) ② 내 문의내역(S3)에 '구성변경' 배지+접수+제목 셀이 일반 셀과 섞인 화면 ③ 문의상세(S4) 구성변경 건 답변대기(접수, "답변을 준비하고 있어요") ④ 화면 전이를 연결선+라벨로 표현: 구독상세[구성변경]→작성폼 / 작성폼[등록하기]→완료시트 / 완료시트[확인]→구독상세 복귀 / S3[구성변경 셀]→S4 / 답변완료 푸시·알림톡→S4(외부진입 라벨만). connector 막히면 화살표+라벨로 대체. 다른 파일 클론이 막히면 무엇이 가능/불가한지 알려라. 로직 주석은 새 규칙(프레임 높이 아래 32px, 겹침 금지)대로.
기술 메모 다른 Figma 파일의 프레임은 직접 클론 불가(Plugin API 단일 파일 제약) → 진입 구독상세는 스크린샷 참조본으로 배치. connector는 design mode 차단 → 화살표+라벨로 대체.
⑤-C 뱃지 색 통일 (2건)
사용자

▶ 사용자 프롬프트 전문 (2건)
[1] (Figma 링크) 여기에 문의리스트 뱃지가 기존과 색상이 달라 '구성변경' 뱃지만 색상을 다르게 만들 이유가 없으니 기존처럼 회색 뱃지로 동일하게 넣어줘 [2] 접수 뱃지는 다른데서는 회색인데 이번에 만든것만 primary야 이것도 맞춰야지
