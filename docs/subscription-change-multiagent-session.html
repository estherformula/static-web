1기획 검토 의뢰 — 블로커·누락 진단

사용자

구성변경 기능개선 초안([개요]/[as-is]/[to-be])으로 지나에게 검토시키고 ⓪폴더 생성 ①블로커 점검 ②부족한 부분 채워 md 저장.

▶ 사용자 프롬프트 전문
에스더팩의 구독상세화면에서 '구성변경' 기능이 있는데 기능 개선을 하기위해 기획을 진행할거야 아래 내용으로 계획을 짰는데 이 기준으로 지나에게 검토시키고 아래 순서로 일을 진행해 0. 스펙리뷰 폴더에 이 건을 진행할 폴더를 하나 만들고 1. 블로커가 있는지 점검 2. 언급되지않은 부족한 부분을 채워서 md파일로 저장 제목 : 1:1문의 기능을 이용한 구성변경 기능개선 [개요] - 구독 상세화면에서 사용자가 직접 구성변경을 요청할 수 있다 - 사용자가 구성변경을 누르면 채널톡이 열리면서 구독을 일시정지하게된다 - 채널톡의 워크플로우는 별도 솔루션에서 진행되므로 진행상태를 알 수 없다 - 사용자가 채널톡에서 액션을 수행하지 않으면 상담원에게 상담이 전달되지 않는다 - 이렇게 되면 구독은 일시정지 되었지만 구성변경 상담은 진행되지 않는 상태가 된다 - 신설된 1:1문의 기능을 이용하면 사용자가 문의를 등록완료하는 시점에 구독을 일시정지 시킬 수 있으니 좀비상태의 구독을 만들지 않아도 된다 [as-is] - 사용자가 구독상세 화면에서 구성변경 버튼 클릭 - 채널톡 워크플로우 오픈 (구성변경 = "780572") - 동시에 '구독일시정지 API' 를 호출하여 구독을 일시정지한다 [to-be] - 사용자가 구독상세 화면에서 구성변경 버튼 클릭 - 1:1문의 입력폼이 나타나며 구성변경 요청내용을 사용자가 입력할 수 있다 - 기존 1:1문의 입력폼과 다르게 '문의 유형', '연결 주문', '제목', '이미지 첨부'가 없다 - 필수입력 : '내용', '알림수단' - 문의등록 API 요청시 구독번호(subscription_number) 를 파라미터에 포함시킨다 - 등록완료시 '구성변경이 요청되었어요.\n완료되면 알려드릴게요.' 토스트가 노출 - 구독 일시정지는 해당 문의가 접수되는 순간 BE에서 처리한다 - 구성변경 1:1문의 유형 타입을 신규로 추가한다 (적절한 이름 제안) - 이 유형은 고객센터의 1:1문의에서는 입력할 수 없다 (선택지에 노출되지 않음)
